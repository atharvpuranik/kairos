# Concepts

**Trace** вЂ” one RAG query: the user's question, the retrieved chunks, the
final answer, latency and token counts. Sent by the SDK, stored 30 days.

**Eval scores** вЂ” computed asynchronously per trace by Claude Haiku:

- *Faithfulness (0вЂ“1)*: does the answer only contain information present in
  the retrieved chunks?
- *Answer relevance (0вЂ“1)*: does the answer address what was asked?
- *Context precision (0вЂ“1)*: what fraction of retrieved chunks actually
  contributed?
- *Hallucination flag + detail*: set when faithfulness is low **and** the
  answer makes specific claims no chunk supports вЂ” with the claim quoted.
- *Failure category*: when any score is below 0.7, the root cause вЂ”
  `chunking`, `embedding`, `reranking`, `prompt`, or `model`.

Scores carry `model_used` and `prompt_version` so they stay comparable as the
eval prompt evolves.

`((faithfulnessГ—0.35) + (relevanceГ—0.35) + (precisionГ—0.30)) Г— 100`, stored as
daily snapshots that power the health chart and week-over-week alerting.

**Alerts** вЂ” created by the worker when the last 24h degrades against your
7-day baseline: faithfulness drop >15% (warning; >30% critical) or a
hallucination-rate spike (>20% and 3Г— baseline). Deduped while unresolved;
resolve from the dashboard.
